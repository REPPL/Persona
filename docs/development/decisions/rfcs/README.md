# Request for Comments

Proposals for future architecture decisions.

## Purpose

RFCs are used for:
- Major architectural changes
- New feature proposals
- Breaking changes
- Community input

## RFCs

*No active RFCs.*

## RFC Process

1. Create RFC document
2. Open for discussion
3. Iterate based on feedback
4. Accept or reject
5. If accepted, convert to ADR

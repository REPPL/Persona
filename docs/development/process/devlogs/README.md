# Development Logs

Thinking logs and development narratives.

## Purpose

Devlogs capture:
- Reasoning behind decisions
- Challenges encountered
- Solutions discovered
- Context for future maintainers

## Format

```markdown
# Thinking Log: [Feature/Fix Name]

**Date:** YYYY-MM-DD

## Context
[What prompted this change]

## Approach Considered
[Options evaluated]

## Decision Made
[What was chosen and why]

## Challenges Encountered
[Problems solved during implementation]

## Outcome
[Result and any follow-up needed]
```

## Naming Convention

`vX.Y.Z-devlog.md` - Version-based naming for milestone devlogs.

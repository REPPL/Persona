## Purpose

Use cases are **Layer 1** of our hybrid specification approach:

```
Layer 1: USE CASES (Why)      ← You are here
    ↓ derive
Layer 2: FEATURE SPECS (What)
    ↓ validate
Layer 3: TUTORIALS (How)
```

Use cases:
- Define user intent without implementation details
- Drive feature specifications
- Provide acceptance criteria for validation

## Use Case Format

Each use case brief includes:
- **Summary** - One-line description
- **User Story** - As a... I want to... so that...
- **Trigger** - What initiates this use case
- **Priority** - P0 (critical), P1 (important), P2 (nice-to-have)
- **Preconditions** - What must be true before
- **Success Criteria** - How we know it's done
- **Derives Features** - Which features implement this
